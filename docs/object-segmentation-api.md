# Object Segmentation API

用于 `OBJECT_DETECTION_PROVIDER=sam` 模式下的点击对象识别 / 分割服务协议。

## 请求

- Method: `POST`
- Content-Type: `application/json`
- Authorization: `Bearer <OBJECT_SEGMENTATION_API_KEY>`（可选）

### Request Body

```json
{
  "image": "data:image/png;base64,... 或 https://...",
  "imageWidth": 1024,
  "imageHeight": 1024,
  "click": { "x": 433, "y": 516 },
  "mode": "point-segmentation"
}
```

### 字段说明

- `image`: 原图，支持 Data URL 或公网 URL
- `imageWidth`: 前端显示坐标系宽度
- `imageHeight`: 前端显示坐标系高度
- `click.x / click.y`: 用户点击点，基于前端显示坐标系
- `mode`: 当前固定为 `point-segmentation`

## 响应

### Success 200

```json
{
  "label": "hat",
  "score": 0.93,
  "bbox": {
    "x": 380,
    "y": 440,
    "width": 180,
    "height": 140
  },
  "polygon": [
    { "x": 392, "y": 451 },
    { "x": 546, "y": 446 },
    { "x": 553, "y": 566 },
    { "x": 401, "y": 578 }
  ],
  "maskUrl": "https://your-mask-host/masks/abc123.png",
  "details": "segment generated by sam2"
}
```

### 字段说明

- `label`: 对象语义标签，可选
- `score`: 置信度，0~1，可选
- `bbox`: 必填，前端目前主要依赖这个做高亮和 fallback
- `polygon`: 可选，推荐返回，用于更精确的轮廓展示
- `maskUrl`: 可选，推荐返回，用于后续真局部编辑
- `details`: 可选，调试信息

## 失败响应

### Error 4xx/5xx

```json
{
  "details": "unable to segment object"
}
```

服务返回非 2xx 时，前端后端链路会自动 fallback 到 bbox 占位方案。

## 坐标约定

所有 `bbox` 和 `polygon` 坐标都基于前端显示坐标系：

- 左上角为 `(0, 0)`
- 最大值不应超出 `imageWidth / imageHeight`
- 不要返回归一化坐标，除非后端先转换回像素坐标

## 推荐实现

如果你后面自己做分割服务，推荐流程：

1. 收到 `image + click`
2. 将图片解码为原始图像
3. 用 SAM / SAM2 / Grounded-SAM / 其他分割模型做 point segmentation
4. 将结果映射回前端显示坐标系
5. 输出 `bbox + polygon + maskUrl`

## 最低可用要求

如果当前服务还不成熟，至少保证返回：

```json
{
  "bbox": {
    "x": 100,
    "y": 100,
    "width": 200,
    "height": 200
  }
}
```

这样前端链路依然能正常工作。
